# 勁揚業務管家 - LINE 官方帳號系統對接與整合說明手冊

本手冊紀錄了 **LINE 官方帳號（機器人）與 Google Apps Script 後台試算表資料庫** 的完整整合設定、功能邏輯以及後續維護步驟。

---

## 🛠️ 一、三大核心功能與自動化機制

### 1. 雙向主動推播通知 (Push Notifications)
系統能在資料發生異動時，精準過濾對象並發送 LINE 私訊：
*   **管理員新增保留**：當在 App/電腦後台建立新的保留項目時，系統會自動比對該店家的負責業務，並將詳細的保留內容直接私訊給該業務的 LINE。
*   **業務扣除/刪除保留**：業務在手機上完成保留結案時，系統會自動私訊通知所有的管理員。
*   **管理員扣除/刪除保留**：管理員刪除某筆保留時，系統會自動私訊通知所屬業務。

### 2. 動態選單切換 (Dynamic Rich Menu)
針對不同身分對象，聊天室下方的選單會自動分流切換：
*   **外部客戶（預設選單）**：
    *   **公司官網**：連線至 `http://www.macarena.com.tw/zh-tw/index.php`
    *   **聯絡客服**：連線至客服小姐的好友邀請連結 `https://line.me/ti/p/w5RBglOOph`
    *   **最新型錄**：開啟雲端硬碟型錄 PDF `https://drive.google.com/file/d/1bmTK4_nWz4Hrs0Qe88yKul-d8VtXfFB1/view?usp=sharing`
*   **內部員工（業務選單）**：
    *   **查詢保留**：發送指令查詢個人未結案的保留。
    *   **查詢庫存**：查詢引導。
    *   **上傳照片**：開啟網頁版 App（PWA）入口進行資料編輯或現場拍照上傳。

### 3. 即時庫存/保留指令查詢 (Chatbot Query)
業務可直接在 LINE 對話框發送指令，系統會即時從 Sheets 資料庫撈取資料：
*   **`查詢保留`**：回傳該業務名下目前所有 open 狀態的保留店家、物品與到期日。
*   **`庫存 [關鍵字]` 或 直接輸入 `[系列與編號]` (如 `EQ-1722` / `KK29`)**：
    *   **智慧搜尋過濾**：若發送的文字在「樣品與展示架」分頁中**有匹配的產品**，機器人會立即回傳型號、數量與存放位置。
    *   **普通閒聊避打擾**：若搜尋**沒有匹配結果**，且您**沒有**輸入「庫存」前綴（例如只傳送一般的聊天文字），機器人會**保持沉默**，確保不會干擾您與別人的正常手動對話。

## 📱 二、業務身分綁定 LINE 流程（一次性設定）

當有新業務加入或需要重新綁定時，請照以下步驟操作：
1.  **發送指令**：在 LINE 聊天室中，對您的官方帳號發送 **`綁定`**。
2.  **開啟網頁**：點選機器人立刻回傳的專屬登入連結。
3.  **登入綁定**：在開啟的網頁中登入該業務的帳號與密碼。
4.  **綁定成功**：登入成功後，後台會自動將該業務的 LINE User ID 寫入試算表的 `Users` 分頁，且**他的手機選單會立刻自動轉變成業務專屬的深藍微光選單**。

## ⚙️ 三、首次安裝與更新設定（管理員必看）

當您更新了後台程式碼，或是需要更新選單上的圖片與連結時，必須執行以下操作：

### 步驟 1：執行 setupLine 建立選單
1.  開啟您的 [Google Apps Script 後台專案](https://script.google.com/home/projects/1vRepq_HNkjbs8vRQvbkkDE8unGPHfksfhOTrkrNZthFZHs2GSHO8Gasc/edit)。
2.  在頂部功能列 of the 函數下拉選單中，選擇 **`setupLine`**。
3.  點擊左側的 **「▷ 執行」** 按鈕。
4.  這會自動向 LINE 註冊建立您的「客戶版選單」與「業務版選單」，並將客戶版設為所有加好友之客人的預設畫面。

### 步驟 2：設定 LINE Webhook 網址
1.  複製您的 Apps Script 網路應用程式 URL：
    `https://script.google.com/macros/s/AKfycbw6p15f3mfeOmnVjvp4niO05J3A_YGMRhmJXqGQ6Jcg_7VQiWZ_4lskjBCZQ2gqbmUKKw/exec`
2.  登入 [LINE Developers 後台](https://developers.line.biz/)，點入您的 Provider ➡️ 選擇該 Bot Channel。
3.  進入 **Messaging API** 頁籤 ➡️ 找到 **Webhook URL**。
4.  貼上網址並儲存，開啟 **Use webhook** 開關，點選 **Verify** 確認顯示成功。

本專案已在 **`v11`** 網頁版更新中徹底拔除了 OneSignal 原生推播服務，不再佔用手機效能：
1.  **後台控制面板**：已刪除「OneSignal 推播設定」表單，避免管理員誤填。
2.  **手機首頁**：已移除「🔔 啟用手機推播」按鈕。
3.  **核心程式**：已從 `service-worker.js` 與 `app.js` 中完整刪除 OneSignal SDK 的引入與 alias 綁定代碼，避免背景喚醒請求。
